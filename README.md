# 🚀 Vue 3 + TailwindCSS + Pinia + TypeScript: Szybki Plan Nauki Fullstack

Celem tego planu jest opanowanie Vue 3, Pinia, TailwindCSS oraz TypeScript w kontekście fullstack developmentu (z wykorzystaniem backendu PHP/Laravel) w około 2 tygodnie.

## 🛠 Stack docelowy

- Vue 3 (Composition API)
- Vite
- TailwindCSS
- Pinia + TypeScript
- Axios / fetch
- Laravel API

---

## 📅 Harmonogram Nauki

### [X] Etap 1: Fundamenty Vue 3 (Dzień 1-2)

- **Cel:** Zrozumienie, jak działa Vue.
- **Tematy:** Composition API (setup, ref, reactive), Template syntax (v-if, v-for, v-model), Eventy (@click), computed vs watch.
- **Mini zadanie:** Todo app (bez stylowania).

### [X] Etap 2: Komponenty + struktura (Dzień 3-4)

- **Cel:** Myślenie komponentowe.
- **Tematy:** props / emits, podział na komponenty, lifecycle hooks (onMounted), slots.
- **Mini projekt:** Appka z listą użytkowników (lista, szczegóły użytkownika, sensowny podział na komponenty).

### [ ] Etap 3: Vite + TailwindCSS (Dzień 5)

- **Cel:** Szybkie UI.
- **Tematy:** setup Vite + Vue, instalacja Tailwind, utility-first CSS.
- **Mini zadanie:** Ostylowanie wcześniejszego projektu Todo / Users list.

### [ ] Etap 4: Routing (Vue Router) (Dzień 6)

- **Cel:** Tworzenie SPA.
- **Tematy:** router, params, layouty.
- **Mini projekt:** 2-3 podstrony (Home, Users, User details).

### [ ] Etap 5: Pinia + TypeScript (Dzień 7-9)

- **Cel:** Zarządzanie stanem aplikacji.
- **Tematy:** store (state, getters, actions), async w store (API), typowanie store i props w TS, typowanie odpowiedzi z API.
- **Mini projekt:** Globalny store (user, lista danych z API).

### [ ] Etap 6: TypeScript w całym projekcie (Dzień 10)

- **Cel:** Pełne typowanie i bezpieczeństwo.
- **Tematy:** props i state, typowanie funkcji i zwracanych wartości, typowanie Pinia store i API responses.
- **Mini zadanie:** Przerobienie dotychczasowych projektów na TS.

### [ ] Etap 7: Real Fullstack Project (Dzień 11-14)

- **Cel:** Spięcie wszystkiego w produkcyjną całość.
- **Projekt:** Dashboard / mini SaaS. Wymagane funkcjonalności: login (mock lub Laravel API), lista danych CRUD, Tailwind UI, Pinia state, TypeScript.
- **Dodatkowo:** Formularze z walidacją, error handling, loading states, deploy na lokalnym serwerze / Vercel / Netlify.

---

## ⚠️ Najczęstsze błędy do uniknięcia

1.  Za dużo teorii, brak kodowania.
2.  Unikanie Composition API i TypeScript.
3.  Brak mini-projektów.

## 💡 Rekomendacje

Każdy etap powinien kończyć się działającym mini-projektem. Wrzucaj swój kod do review i na bieżąco go poprawiaj (z mentorem lub samodzielnie).

**Wynik końcowy:** Po ukończeniu planu będziesz w stanie budować nowoczesne aplikacje fullstack z wykorzystaniem Laravel + Vue 3 + TS + Pinia + TailwindCSS.
